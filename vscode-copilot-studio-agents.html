Editing & Publishing Copilot Studio Agents with VS Code

A practical, code-first workflow for authoring Microsoft Copilot Studio agents in Visual Studio Code — pull, edit, source-control, and publish. Prepared as an enablement reference · July 2026.

The one thing to remember: The Copilot Studio web maker is best for prototyping and visual editing; Visual Studio Code shines once an agent matures — bringing source control, diffs, bulk edits, and CI/CD to the agent definition. The two are complementary, not either/or.

1. Why author agents in VS Code?

The browser-based maker experience is fast and visual, but a code-first workflow unlocks capabilities the browser can't match:

  • Source control — commit agent definitions to Git, review changes in pull requests, and roll back cleanly.
  • Bulk edits — find-and-replace across topics, instructions, and prompts in seconds.
  • Diff & review — see exactly what changed between versions.
  • Team collaboration — multiple makers working against the same agent with real merge history.
  • CI/CD — promote agents across Dev → Test → Prod with pipelines instead of manual export/import.

3. Set up the tooling & connect

1

Install the extension

Open VS Code → Extensions (Ctrl+Shift+X), search for "Power Platform Tools", and install it. Confirm the bundled CLI is available:

pac --version
2

Authenticate to your environment

Create an auth profile pointing at your Power Platform environment, then list or switch profiles as needed:

pac auth create --environment https://yourorg.crm.dynamics.com

pac auth list
pac auth select --index 1
3

Bring the agent local

Copilot Studio agents live inside a Power Platform solution. Export the solution and unpack it into source files you can edit and place under version control:

# See which agents exist in the environment
pac copilot list

# Export and unpack the solution that contains the agent
pac solution export --path ./myagent/MyAgentSolution.zip --name MyAgentSolution
pac solution unpack --zipfile ./myagent/MyAgentSolution.zip --folder ./myagent-src
Tip: Commit the freshly unpacked agent to Git before editing. That clean baseline is what every future diff compares against.

4. Edit the agent

Once unpacked, the agent is just text — so all of VS Code's editing power applies (multi-file search, snippets, extensions, Git diffing). Inside the source you can modify:

Conversation & behavior

  • Topics & conversation flows — authored in YAML.
  • Agent instructions / system prompts — the natural-language behavior guiding generative answers.

Data & capabilities

  • Knowledge sources & connections — references to the data the agent draws on.
  • Actions & tools — connectors and flows the agent can invoke.

5. Publish your changes

Repack the source into a solution, import it back into the environment, then publish:

pac solution pack --zipfile ./myagent/MyAgentSolution.zip --folder ./myagent-src
pac solution import --path ./myagent/MyAgentSolution.zip

Then open the agent in Copilot Studio (web) and select Publish to push the changes live to your channels — or automate publishing as part of a pipeline.

6. Recommended workflow

Web maker (prototype)  →  Pull to VS Code  →  Edit + commit to Git
      →  PR review  →  Import to Test  →  Validate  →  Import to Prod  →  Publish
Prototype in web Refine in code Review via PR Promote by pipeline

7. Good practices

  • One solution per agent where practical — keeps import/export clean.
  • Use managed solutions downstream (Test/Prod); unmanaged in Dev.
  • Never hard-code secrets — use environment variables and connection references.
  • Automate with pipelines — Power Platform Pipelines, or Azure DevOps / GitHub Actions driving the pac CLI.
  • Test after every import — validate topics and generative responses before you publish.
A note on scope: Code-first authoring for Copilot Studio is evolving quickly. The pac solution export/unpack path above is the reliable, generally-available method today; some preview capabilities add more direct agent pull/push commands. Always confirm the current command surface against the Microsoft Learn references below before standing up a pipeline.
