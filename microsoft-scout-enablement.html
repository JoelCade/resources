Enabling Microsoft Scout for Your Organization

A step-by-step admin guide to turning on Microsoft Scout for users in a customer (non-Microsoft) tenant — the prerequisites, the four enablement gates, and how model access is licensed. Prepared as an enablement reference · July 2026.

The one thing to remember: Installing the Scout app does not grant access on its own. A user can sign in only after all four enablement gates below are complete: tenant enablement, organization attestation, device policy, and licensing. If any gate is missing, the user sees a waitlist screen or a failed sign-in — and the cause is not always obvious inside the app.

Prerequisites at a glance

RequirementDetail
IdentityEach user needs a Microsoft 365 work or school (Entra ID) account.
Model licensingEach end user needs a GitHub Copilot Business or Enterprise license. Scout currently uses GitHub Copilot licensing and token infrastructure for model access.
Admin rolesAccess to the Microsoft 365 admin center (Copilot settings) and the Microsoft Intune admin center (device configuration).
Managed devicesTarget devices should be enrolled in Intune (or your MDM) so the Scout access policy can be applied.

How AI model access is provided

Scout does not bring its own separate token pool for customer organizations today. Instead:

  • Model access is provisioned through GitHub Copilot. Users must hold a GitHub Copilot Business or Enterprise seat.
  • Consumption is billed against GitHub Copilot AI credits / tokens associated with that seat.
  • During first-run setup, each user associates Scout with their GitHub account, which links their Copilot entitlement to the app.
Plan for two licenses per user: a Microsoft 365 (Entra) identity and a GitHub Copilot Business/Enterprise seat. Confirm current SKU requirements and credit/consumption terms with your Microsoft account team before rollout — these evolve during the program.

The four enablement gates

1

Enable Copilot Frontier in the Microsoft 365 admin center

  1. Sign in to the Microsoft 365 admin center.
  2. Go to Copilot → Settings → View all, then search for Frontier and open Copilot Frontier.
  3. Set the access scope: No access, All users, or Specific users (recommended for a pilot — add just the intended user or group).
  4. Save the configuration.
Allow time to propagate. Enablement can take several hours to take effect after you save.
2

Complete the organization attestation / sign-up

Complete the Frontier organization sign-up / attestation form for your tenant. Because Scout may route data through GitHub-related inference paths, the organization must explicitly opt in and attest before access is granted. This is a one-time, tenant-level step.

3

Allow Scout access on managed devices via Intune

Obtain the Scout device-policy files from the Scout admin repository:

microsoft-scout.admx microsoft-scout.adml microsoft-scout.mobileconfig

Windows

  1. Import the ADMX / ADML templates into Intune.
  2. Create a Windows configuration policy using Imported Administrative Templates.
  3. Set Allow Microsoft Scout Frontier access = Enabled.
  4. Assign the policy to your pilot devices (or all devices).

macOS

  1. Create a macOS Custom profile.
  2. Upload the microsoft-scout.mobileconfig file.
  3. Deploy through the Device channel and assign to target devices.
4

Assign licenses to the user

  • Ensure the user has a Microsoft 365 work/school account.
  • Assign a GitHub Copilot Business or Enterprise license to the user (this is what provides model access and token consumption).

Final step — the user signs in

Only after gates 1–4 are complete:

  1. The user installs the Microsoft Scout application on a managed device.
  2. The user signs in with their Microsoft 365 (Entra) account.
  3. During first-run setup, the user links their GitHub account to associate their Copilot entitlement with Scout.
Troubleshooting a blocked sign-in: if a user is stuck on a waitlist screen or sign-in fails, work back through the gates — the most common causes are (a) Frontier not yet propagated, (b) the attestation form not submitted, (c) the Intune access policy not applied to that device, or (d) a missing GitHub Copilot license.
