Personal AI Operating System · Enablement

The Second Brain & the Chief of Staff

How I run my day with an LLM-maintained knowledge vault and an agent "agency" — a durable memory plus a runtime that filters, decides and routes. · July 2026

The one thing to remember Give the AI a durable memory — and let it run like a team. Chat sessions are ephemeral; a versioned personal wiki is not. Pair that vault with a Chief-of-Staff operating mode and daily automations, and the work of each day becomes durable knowledge automatically — even while you sleep.

The problem: an SE's knowledge evaporates

Real working context lives scattered across a dozen surfaces, and the tools that hold it have no memory of what happened yesterday.

Scattered

Context is everywhere

Email, Teams, meeting recaps, CRM, file storage — no single place that actually knows my accounts.

Ephemeral

Sessions start cold

Every AI chat begins from zero. Insights from last week vanish the moment the window closes.

Lost

Nothing becomes durable

Recaps get read once. Decisions, deal facts and action items quietly slip away.

The two connect through a simple discipline: reads stay open, writes are disciplined. The runtime reads the brain freely; only one path writes back to it. Daily automations keep the brain fed so each day's work turns into notes on its own.

The Second Brain: an LLM-maintained personal wiki

A durable wiki (the "build a Second Brain" pattern). Chat is ephemeral; the vault is the memory that persists.

PARA-structured

Inbox → Notes → Projects → Areas → Resources → Archive. Entities and concepts, one idea per page.

Git-versioned

Every change is committed. Full history, nothing silently lost, a clean tree after each operation.

One writer, private

A single agent is the sole writer; the vault lives in my own storage and is never shared with third parties.

Raw + refined

Immutable source records sit alongside distilled, cross-linked wiki notes — reusable reference assets.

How it's organised — three layers, knowledge flows upward

LayerWhat it holdsWho maintains it
RawPastes, clippings, transcripts, quick captures. The immutable ground truth — never edited or deleted.Human writes · agent appends only
WikiDistilled, atomic, cross-linked notes with [[wikilinks]]. The refined brain, derived from raw.Agent maintains
SchemaOperating manual, index, log, templates and maps — the rules that make the vault self-describing.Agent maintains · human governs
Golden rule Raw is ground truth and is never rewritten. The wiki is derived from raw and may be rewritten freely. Schema describes both.

Why the vault stays trustworthy

Full frontmatter

Title, type, created, updated, tags, sources, status — on every page.

One idea per page

Kebab-case filenames; the filename is the page's stable identity.

Links over duplication

Wikilinks connect entities and concepts instead of copying content.

Cite sources, flag doubt

Every claim traces to a raw record or a URL; unverified items are marked as such.

Log every change

An append-only, greppable log records each operation with a dated tag.

Commit each operation

A commit script leaves a clean tree — one operation, one commit.

The Chief of Staff: the default operating mode

The runtime works every session by default — not as a chatbot, but as the person who keeps the desk clear and the standards high.

Filter the noise

Name the 1–3 things that actually matter right now; suppress the rest.

Reduce decisions

Recommend a single next move — headline first, then the one-line why.

Route, don't do

Design, research, comms, data, ops, eng — delegated to specialist teams.

Hold standards

Voice, brand, sensitivity and privacy enforced on every artifact.

Auto plan mode Anything larger than ~3 steps, or touching multiple files, tools or people, flips to a written plan — headline plus todos — and waits for an explicit go before executing. State (plan, todos, memory) stays current so work survives sessions.

The agency: a team of specialist sub-agents

The runtime is the router. Teams never call each other — each emits a handoff, and the router decides the next hop.

/librarian

The vault write-path.

Rules

Max 3 hops then check in · confirm every outbound send · security can enter any chain.

Governance: one writer, one set of rules

Reads are open to every team. Writes funnel through a single path so conventions, provenance and commit discipline always hold.

1

Any team produces durable knowledgeDesign, research, ops — anyone can generate something worth keeping.

2

The runtime decides what's durableThe Chief of Staff picks what's worth keeping; a handoff goes to the librarian.

3

The Librarian is the sole write-pathOne agent creates, edits, moves and commits vault files — no other team writes directly.

4

A privacy gate before diskElevated-sensitivity content is never written in plain text — it routes to security first.

5

Index + log, then commitThe index and log are updated and the commit script seals it into git with a clean tree.

While I work — and while I sleep

Daily automations keep the vault fed. Scheduled agents sort, capture, ingest and consolidate — turning each day's work into durable notes automatically.

Deep dive · the meeting-brief ingest

From a meeting to a memory, in five moves:

1

Meeting endsThe daily brief picks up today's completed customer meetings.

2

Preserve rawThe recap is saved verbatim to the raw layer — dated and immutable.

3

Distill to wikiDurable facts become entity & concept pages with wikilinks.

4

Index + logThe index and log are updated so it's discoverable.

5

CommitThe commit script seals the operation into git history.

Real example A customer hackathon recap is ingested → the account's entity page is updated with a deal-risk flag, plus partner contacts on a related page. Raw preserved, sources cited, committed — all without a manual step.

Deep dive · nightly consolidation

Each weeknight the dream loop reviews the last 24 hours and decides what deserves to be remembered — keeping the memory lean and true, not padded.

Score the day

Weighs candidate facts from the last 24h of sessions.

Promote the durable

Only high-confidence, novel facts reach permanent memory.

Dedupe the rest

Recurring themes fold into what's already known — no near-duplicates.

Write the diary

A short reflective entry; never raw email or chat content.

It all runs on a loop

A day, end to end:

  • Morning — a web-grounded industry brief lands in the vault.
  • Through the day — the heartbeat sorts the inbox, flags asks and captures tasks; customer recaps are generated as I go.
  • Late afternoon — the daily brief distils today's recaps into notes.
  • Night — the dream loop consolidates durable facts into memory.
The loop Work generates signal → automations capture & distil → the vault grows → tomorrow's context starts richer.
